## フロントエンドアプリの種類と構造（階層構成）

👉レンダリング方式でいくつか分岐するけど、画面遷移は結局「JSルーティング」。全員同じSPA。

- FEアプリの種類
  - MPA (Multi Page Application)
    - ページごとにサーバーから HTML を取得
  - SPA (Single Page Application)
    - **レンダリング方式（初回描画）**
      - CSR (Client-Side Rendering)
        - 初回 HTML は最小限、ブラウザで JS を実行して DOM を生成
      - SSR (Server-Side Rendering)
        - 初回 HTML をサーバーで生成して返却、即表示。gSSPはこれ✅
      - SSG (Static Site Generation)
        - ビルド時に HTML を生成して配布、リクエスト時は静的 HTML を返却
    - SPAフレームワーク
      - React, Vue, Angular, Svelte
    - Metaフレームワーク
      - Next.js, Nuxt.js, Remix
    - SPAの種類・進化
      - モダンSPA
        - ハイブリッドレンダリング対応（初回SSR or SSG + 以降CSR）
      - LegacySPA
        - 初回から CSR のみ
