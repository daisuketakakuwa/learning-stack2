## フロントエンドアプリの種類を決める
```
FEアプリの種類（UI構造）
  │
  ├─ MPA
  │    └─ 基本はリクエスト時サーバー生成（SSR）
  │
  └─ SPA
        ├─ Legacy SPA
        │    └─ 実質CSRのみ
        │
        └─ Modern SPA
              ├─ HTML生成戦略
              │     ├─ リクエスト時生成
              │     │     ├─ CSR
              │     │     └─ SSR
              │     │
              │     └─ ビルド時生成
              │           └─ SSG
              │
              ├─ 比較観点
              │     ├─ 初回描画性能
              │     ├─ 画面遷移性能
              │     ├─ SEO
              │     └─ サーバー負荷
              │
              ├─ ホスティング方式（戦略に依存）
              │     ├─ 静的ホスティング（CSR / SSG）
              │     └─ Nodeサーバー（SSR）
              │
              └─ SPAフレームワーク > Metaフレームワーク
                    │
                    ├─ React
                    │     └─ Next.js
                    │
                    ├─ Vue.js
                    │     └─ Nuxt.js
                    │
                    ├─ Angular
                    │     └─ （Angular Universal など）
                    │
                    └─ Svelte
                          └─ （SvelteKit など）
```
