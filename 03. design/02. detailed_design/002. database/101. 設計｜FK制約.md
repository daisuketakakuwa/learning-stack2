## 子テーブル側の `client_id` に参照制約を設定
### RESTRICT
親テーブルに紐づけて履歴・証跡 のために残しておきたいとき
```sql
CONSTRAINT clients_fk FOREIGN KEY (client_id) REFERENCES public.clients (id)
  ON DELETE RESTRICT ✅ここの設計が大事
  ON UPDATE RESTRICT 🔴ID変更は基本「ない前提」でOK
```
### CASCADE
親の動きに合わせる（親消えるなら子も消える、親変わるなら子も変わる）
```sql
CONSTRAINT clients_fk FOREIGN KEY (client_id) REFERENCES public.clients (id)
  ON DELETE CASCADE ✅ここの設計が大事
  ON UPDATE CASCADE 🔴ID変更は基本「ない前提」でOK
```

### SET NULL
親変わる/消える → 子をnullにする　👉関係は切れるが、分析等の用途でデータ自体は残したい
```sql
CONSTRAINT clients_fk FOREIGN KEY (client_id) REFERENCES public.clients (id)
  ON DELETE SET NULL ✅ここの設計が大事
  ON UPDATE SET NULL 🔴ID変更は基本「ない前提」でOK
```
