## このリポジトリで知識をどう整理しているか
|ディレクトリ|役割|
|---|----|
|01. process|仕事としての開発|
|02. principles|変わらない基礎|
|03. design|設計。この中でも主に２種類に分類<br>└`architecture`：構造/責務/境界を決める<br>└`detailed_design`：具体的な実装を決める|
|04. stack|実装するための具体技術|

### 02. principleは「概念・知識」 「使い方」ぽくなった瞬間に 04. stack に落ちる。
`02. principles` は<br>
・「どう作るか」ではなく<br>
・「何者なのか/どういう概念なのか」<br>
を扱いたいので、「使い方」「実装方法」っぽくなった瞬間に `04. stack` でまとめる。
